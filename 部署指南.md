# NameCloud 部署指南

## 方式一：Vercel（推荐，免费，最快）

[Vercel](https://vercel.com) 是最简单的部署方式，支持自动 HTTPS，全球 CDN 加速。

### 步骤：
1. **注册账号**
   - 打开 https://vercel.com
   - 点击 "Sign Up" → 选择 "Continue with GitHub"（或 GitLab / 邮箱注册）

2. **安装 Git 并上传代码**
   ```bash
   # 在项目目录打开终端
   cd d:\vibecoding\namecloud
   
   # 初始化 Git
   git init
   git add .
   git commit -m "init"
   ```

3. **在 GitHub 创建仓库**
   - 打开 https://github.com/new
   - 仓库名填 `namecloud`
   - 创建后按提示关联本地仓库：
   ```bash
   git remote add origin https://github.com/你的用户名/namecloud.git
   git push -u origin main
   ```

4. **在 Vercel 导入**
   - 打开 https://vercel.com/new
   - 选择刚才的 GitHub 仓库
   - 框架选 "Other"
   - 点击 "Deploy" → 等待几秒
   - 部署完成会得到一个链接：`https://namecloud.vercel.app`

5. **配置 API Key（可选）**
   - 在 Vercel 项目设置中 → Environment Variables
   - 添加 `DEEPSEEK_API_KEY` = 你的 Key
   - 重新部署即可

### 优点：
- ✅ 免费（每月 100GB 带宽）
- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 关联 GitHub 自动部署

## 方式二：Netlify（同样免费简单）

1. 注册 https://netlify.com
2. 点击 "Add new site" → "Import an existing project"
3. 连接你的 GitHub 仓库
4. 部署设置：
   - Branch: `main`
   - Publish directory: 留空（因为是根目录）
   - 点击 "Deploy site"
5. 完成！得到 `https://xxx.netlify.app`

---

## 方式三：GitHub Pages（纯免费，无需注册第三方）

1. 在 GitHub 创建仓库 `你的用户名.github.io`
2. 上传代码到 main 分支
3. 打开仓库 Settings → Pages
4. Source 选 "Deploy from a branch"
5. Branch 选 `main`，目录选 `/ (root)`
6. 等待几分钟，访问 `https://你的用户名.github.io`

## 方式四：自己的服务器（Nginx）

如果你有自己的云服务器：

1. **安装 Nginx**
   ```bash
   # Ubuntu/Debian
   sudo apt update
   sudo apt install nginx
   
   # CentOS
   sudo yum install nginx
   ```

2. **上传文件**
   ```bash
   # 从本地复制到服务器
   scp -r d:\vibecoding\namecloud\* 用户名@服务器IP:/var/www/namecloud/
   ```

3. **配置 Nginx**
   ```bash
   sudo nano /etc/nginx/sites-available/namecloud
   ```
   写入：
   ```nginx
   server {
       listen 80;
       server_name 你的域名或IP;
       
       root /var/www/namecloud;
       index index.html;
       
       location / {
           try_files $uri $uri/ =404;
       }
   }
   ```

4. **启用并重启**
   ```bash
   sudo ln -s /etc/nginx/sites-available/namecloud /etc/nginx/sites-enabled/
   sudo nginx -t  # 测试配置
   sudo systemctl restart nginx
   ```

## 部署前的重要配置

### 1. 填写 API Key
打开 `config.js`，把 `sk-你的APIKey` 替换为你的真实 DeepSeek API Key：
```javascript
DEEPSEEK_API_KEY: 'sk-你的真实Key',
```

### 2. 可选：改用环境变量（更安全）
如果部署到 Vercel/Netlify，可以修改 `ai.js` 从环境变量读取 Key：
```javascript
// 在 ai.js 中，将 CONFIG.DEEPSEEK_API_KEY 替换为：
const apiKey = CONFIG.DEEPSEEK_API_KEY || process.env.DEEPSEEK_API_KEY;
```
